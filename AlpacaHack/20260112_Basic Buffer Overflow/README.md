# Basic Buffer Overflow

## / Overview

アドレスの相対位置を使ったROP

## / Writeup

今回のチャレンジは先月の[simpleoverwrite](../20251227_simpleoverwrite/)と同じ発想でクリアできる。

異なる点は、関数のアドレスが実行するたびに変わるということ。しかし、main関数のアドレスは実行する度に与えられるので、そこからwin関数のアドレスが計算できる。

具体的には、win関数とmain関数の相対位置は変わらないので、配布されている実行ファイル `chal` からそれぞれの関数のアドレスを取得する。ここから

`実際のwinアドレス` = `実際のmainアドレス` - `chalのmainアドレス` + `chalのwinアドレス`

が計算できる。後は配列のサイズが 64 であることから、適当な 72 バイトに`実際のwinアドレス`を結合して送信することで、サーバのシェルが得られる。
